两路camera如何分别控制

问题：两路camera是如何实现不同的参数的？

需要看HAL层camera类的构造函数。可以看到HWI中是总的类，里面包含状态机，回调函数等等函数封装，初始化用QCamera2HardwareInterface（camera_id)，多一个摄像头就增加一个对象。

在hal层里面加一个对cameraid的判断，由于是创建了两个对象，互不干扰，因此只需要对id号进行判断，赋值还是一样的赋值。

注意了ispctrl参数是用QCameraParameters进行传递，它是在QCameraParametersIntf类里面的，随着QCamera2HardwareInterface类的定义而定义。QCamera2HardwareInterface又是在QCamera2HardwareInterface中定义的，也就是说随着一个摄像头的创建，会创建这个摄像头的整个实现方法。

C++中的static变量

静态数据成员

　　在类中，静态成员可以实现多个对象之间的数据共享，并且使用静态数据成员还不会破坏隐藏的原则，即保证了安全性。因此，静态成员是类的所有对象中共享的成员，而不是某个对象的成员。

但是遇到一个问题，需要保存上一个动作的时间，以前都是用static的方法保存，但是用对象的方法时，就不能这样了，创建一个新的对象，这个时间不能自动增加一个。就会导致异常！！

因此static变量要注意使用的时机。

解决方案：将static变量转成私有的类成员变量，既能保证私有性，又能保证创建新对象的时候自动新建变量。

找到原因

EEPROM 的HID问题

我用的8076平台的HID，而660平台的HID是0x368

系统提供了一个更新这个E2信息的命令
生产工具也可以烧E2的，我们使用前都要把它烧成660的PID和HID

hw_test pinfo update -p 0 -i 0x0368 -d 0x1565 -v 0 -s 1 -m 20170712 -o 0 -t 0 -a 0 -f 0 -q 0 -e “07:14:45:07:07:78” -w 0 -b 0 -u “8056_IMX185” -n 1

其中07:14:45:07:07:78是MAC地址，每换一个设备要换一个，在hwinfo中可以读取

修改方案：

hal:修改pid_hid_list.h

vendor: 修改xml

ispctrl:修改pid_hid_list.h

依旧起不来

  1. 因为我推得动态库，其中还修改了xml文件。编译system.img推进去
  2. xml对HID的选择正确，但是还是起不来。通过log发现有一个动态库没有找到，导致摄像头异常。
  3. 由于APK在system.img里面，重新编译就可能业务起不来，因此重新安装了一个APK

c语言随笔

c++ new和delete

new一个自定义类型 ClassA 的对象时，C++ 会先调用 malloc 来申请一块 sizeof(ClassA) 大小的内存（操作系统会记录这块内存的首地址与大小），然后调用 ClassA 的构造函数在这块内存上初始化对象。此时，new 关键字会返回 malloc 得到的地址。调用delete时，会首先执行 ClassA 的析构函数，再调用 free 释放 malloc 得到的指针。

由于void指针不存在析构函数，所以任何时候都不要尝试 delete void 指针。
